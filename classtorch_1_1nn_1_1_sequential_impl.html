Detailed Description

A Sequential module is a container for any number of other modules. Its forward() method chains outputs to inputs and returns the final output. The Sequential class reference semantics.

template<typename... Modules>
torch::nn::SequentialImpl::SequentialImpl ( Modules &&...  modules)
inlineexplicit

Constructs the Sequential from a pack of modules. Each module can either be a plain value (e.g. Linear) or a boxed value (e.g. shared_ptr<Linear>). Unboxed modules will be moved into shared_ptrs internally.

◆ push_back() [2/6]

template<typename M , typename = torch::detail::enable_if_module_t<M>>
void torch::nn::SequentialImpl::push_back ( M &&  module)
inline

Adds a new Module to the Sequential container, moving or copying it into a shared_ptr internally. This method allows passing value types, and letting the container deal with the boxing. This means you can write Sequential(Module(3, 4)) instead of Sequential(std::make_shared<Module>(3, 4)).
