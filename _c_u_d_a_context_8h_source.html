16 /*
17 A common CUDA interface for ATen.
18 
19 This interface is distinct from CUDAHooks, which defines an interface that links
20 to both CPU-only and CUDA builds. That interface is intended for runtime
21 dispatch and should be used from files that are included in both CPU-only and
22 CUDA builds.
23 
24 CUDAContext, on the other hand, should be preferred by files only included in
25 CUDA builds. It is intended to expose CUDA functionality in a consistent
26 manner.
27 
28 This means there is some overlap between the CUDAContext and CUDAHooks, but
29 the choice of which to use is simple: use CUDAContext when in a CUDA-only file,
30 use CUDAHooks otherwise.
31 
32 Note that CUDAContext simply defines an interface with no associated class.
33 It is expected that the modules whose functions compose this interface will
34 manage their own state. There is only a single CUDA context/state.
35 */
